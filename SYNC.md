# Developing on Two Laptops (Windows + Mac)

GitHub is the sync hub. Both laptops push to and pull from the same repo:
`https://github.com/anjanikmaurya/SWAXS_Autonomous_Platform_v1.git`

## The one habit that avoids all problems

**Pull before you start. Push before you stop.**

```bash
git pull                              # get the other laptop's latest work
# ...develop...
git add -A
git commit -m "what you changed"
git push                              # send it up
```

As long as you never edit both laptops at the same time without pushing in
between, you will never hit a merge conflict.

## First-time setup (already done once, keep for reference)

Line endings are normalized by `.gitattributes` (LF in the repo, native per OS).
This prevents the "every file looks modified" problem when switching between
macOS (LF) and Windows (CRLF). No action needed day to day.

## If you forgot to push and edited both laptops

```bash
git pull                              # git will try to merge
# if it reports conflicts, open the listed files, fix the <<<< ==== >>>> marks,
git add -A
git commit
git push
```

## Things NOT tracked in git (rebuilt locally on each machine)

- `.venv/`, `venv/` — Python environment (recreate with `uv`)
- `ai_knowledge/vector_db/` — AI knowledge vector database (regenerated by the assistant)
- `.env` — secrets (copy from `.env.example` and fill in per machine)
- data folders (`Demo_Data/`, `atT/`, etc.)

After a fresh clone or pull on a new machine:

```bash
uv sync                               # or: uv run <app>  (recreates the env)
cp .env.example .env                  # then fill in secrets
```

## Quick reference

| Situation                    | Command                                   |
|------------------------------|-------------------------------------------|
| Start work on a laptop       | `git pull`                                |
| Save + share your work       | `git add -A && git commit -m "..." && git push` |
| See what changed             | `git status`                              |
| Discard local uncommitted mess | `git checkout -- .`                     |
